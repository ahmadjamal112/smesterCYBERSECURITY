Authentication

Authentication is the process of verifying the identity of a user before allowing access to a system. It checks that you are who you claim to be, using credentials like passwords, biometrics, or tokens. Without authentication, anyone could enter your system.


‚Üí VERIFY ‚Üí IDENTIFY ‚Üí GRANT TOKEN

// Answers: "WHAT CAN YOU DO?"

Access Control

Access Control decides what an authenticated user is allowed to do within a system. Even after you're logged in, access control ensures you can only reach resources you're permitted to ‚Äî protecting sensitive data from unauthorized users.


‚Üí CHECK ROLE ‚Üí ENFORCE POLICY ‚Üí ALLOW/DENY

// Access Control Models

Three Core Models

Different organizations use different models depending on their security needs and structure.

RBAC

Role-Based Access Control

Access is assigned based on the user's role in the organization. Users with the same role get the same permissions. Easy to manage at scale.

Admin Manager User

Used in: Hospitals, Universities, Enterprises

DAC

Discretionary Access Control

The resource owner decides who can access their files or data. You can share files with specific people ‚Äî like Google Docs sharing.

Owner Guest Editor

Used in: File systems, Google Drive, Dropbox

MAC

Mandatory Access Control

The system enforces access policies ‚Äî not users. Data has sensitivity labels (Top Secret, Confidential). Users can only access what matches their clearance.

Top Secret Secret Public

Used in: Military, Government, Intelligence

Research Paper

A summary of the selected academic paper on Authentication and Access Control mechanisms.

Authentication and Access Control Mechanisms in Computer Systems
Published in IEEE / International Journal of Computer Security | Source: Google Scholar

Abstract: This research paper explores the critical importance of authentication and access control in protecting modern computer systems from unauthorized access and cyber threats. The paper presents a comprehensive review of authentication mechanisms, including password-based authentication, biometric systems, and multi-factor authentication (MFA).

The study examines three primary access control models ‚Äî Role-Based Access Control (RBAC), Discretionary Access Control (DAC), and Mandatory Access Control (MAC) ‚Äî analyzing their effectiveness, limitations, and appropriate use cases across different organizational environments.

Findings indicate that combining strong multi-factor authentication with properly implemented access control policies significantly reduces the risk of data breaches and cyberattacks. The paper concludes with recommendations for organizations to adopt layered security models and continuously audit their authentication and authorization mechanisms to adapt to evolving threats.

// Lab Questions & Answers

Student Responses

Q1. What is TryHackMe and why is it useful?

TryHackMe is a free, browser-based cybersecurity learning platform that provides virtual labs, guided rooms, and capture-the-flag (CTF) challenges. It is useful because students can practice real ethical hacking techniques in a safe, legal, controlled environment without needing to set up their own lab infrastructure.

Q2. What is Offensive Security?

Offensive security is the proactive practice of thinking and acting like an attacker to identify vulnerabilities in systems before real malicious hackers can exploit them. It includes penetration testing, vulnerability scanning, and ethical hacking ‚Äî all performed with explicit authorization from the system owner.

Q3. What was the purpose of scanning the website?

The purpose was to discover hidden or unlinked pages that are not visible through normal browsing. Tools like GoBuster automate this by testing thousands of common directory names against the web server, revealing pages the developers may have forgotten to secure.

Q4. What risk does an exposed hidden page create?

An exposed hidden page ‚Äî especially one with sensitive functionality like bank transfers ‚Äî creates a critical security risk. Any attacker who discovers it can perform unauthorized actions, steal data, or manipulate transactions without ever needing a password. This violates Confidentiality and Integrity from the CIA triad.

Q5. What did you learn from this lab?

This lab demonstrated that security through obscurity (hiding pages without protecting them) is not real security. Just because a page has no visible link does not mean it cannot be found. Proper authentication and access control must be enforced on every sensitive endpoint regardless of how "hidden" it seems.
